# 开发完成审查

审查日期：2026-06-21

## 结论

本地 MVP 已完成并冻结为基线。运行时统一为 MySQL 8；Docker Compose 会启动 MySQL、Go 后端、Nginx 前端和一次性验证容器；MySQL 不映射宿主机端口，只有前端映射到 `127.0.0.1:18080`。

2026-06-20 后续联调补齐了可体验的登录边界：首次启动创建或初始化 `admin` 管理员，Web 使用用户名密码登录并由 HttpOnly JWT Cookie 维持会话。匿名业务 API 返回 401；管理员可进入基础设施、策略和管理页面，发布动作一律以当前会话用户记录，不能再由前端传入任意用户 ID 冒充操作者。

本轮前端使用体验优化已完成：顶部导航按日常发布、管理员配置和个人访问密钥收敛；工作台只保留待处理任务，管理员首次使用时根据实际配置状态展示三步发布准备清单；发布中心、创建发布、发布记录和发布详情支持 URL 深链及浏览器前进/后退恢复。普通用户手动访问管理员路径会返回工作台。

本轮已通过 `make verify` 与隔离 Compose 空库功能检查。后者从空 MySQL 数据库自动迁移并覆盖 Mock 发布的驳回、取消、成功、服务器组、partial/skipped、失败、重新发布、回滚、日志和事件流。重新发布始终创建新的发布单，重新执行预检与确认，不会篡改原失败记录。

浏览器人工验收已完成。验收中发现 `127.0.0.1:18080` 被本项目遗留的宿主机 Go 服务进程占用，导致页面返回后端 404；停止该进程并重建 `web` 容器后，地址恢复为 Nginx `200 OK`。随后在浏览器中完成初始化 Mock 配置、创建发布单、确认入队、等待执行成功、进入发布记录并查看服务器日志；成功发布记录显示 `success`，服务器日志显示对应 Mock 服务器为 `success`。
