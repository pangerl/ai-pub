# 前端信息架构设计

## 1. 目标

第一版前端是 React 最小管理界面，目标是让员工和管理员通过 Web 完成发布闭环。

原则：

- 页面围绕用户任务组织。
- 可见文案默认中文。
- 不按数据库表机械堆页面。
- 不做营销页和复杂大屏。
- 页面视觉、色彩、排版和组件状态以同目录的 `DESIGN.md` 为唯一规范；信息架构本文只定义用户任务与交互边界。

## 2. 页面优先级

P0 页面必须支持第一版发布闭环。

P1 页面需要在导航和信息架构中预留位置，但可以按开发计划后置。

## 3. 导航结构

```text
工作台
发布
  发布中心
  创建发布单（主操作）
  发布记录
配置（管理员）
  基础设施
策略（管理员）
  发布策略
系统（管理员）
  用户与权限
  集成访问密钥       # P1
  通知与凭据         # P1
账户操作
  个人访问密钥       # P1
```

## 4. P0 页面

### 4.1 工作台

目标：给用户进入发布工作的入口。

内容：

- 待我确认。
- 我发起的运行中发布。
- 最近失败。
- 基础入口：创建发布单、查看发布中心、查看发布记录。

说明：

- P0 工作台只要求提供发布处理入口和关键列表摘要。
- 工作台默认优先展示待处理任务，不把统计、趋势和分布与任务入口混排。
- 首次管理员进入且缺少可发布配置时，显示项目/服务/版本、运行环境、部署连接三步检查清单。
- 更完整的趋势、环境分布、队列堆积等按 P1 另设运行概览，不占用日常任务入口。

### 4.2 发布中心

目标：统一承载待确认、我发起、全部发布单。

视图：

- 待我确认。
- 我发起。
- 全部发布。

筛选：

- 状态。
- 服务。
- 环境。
- 申请人。
- 来源。
- 时间。

列表字段：

- 发布单号。
- 服务。
- 环境。
- 版本。
- 状态。
- 申请人。
- 来源。
- 更新时间。
- 下一步动作。

### 4.3 创建发布单

流程：

```text
选择服务
  -> 选择环境
  -> 选择版本
  -> 选择部署目标
  -> 执行 preflight
  -> 补充可选上下文
  -> 创建发布单
```

要求：

- 缺失服务、环境、版本、部署目标时不能提交。
- 普通发布不强制填写发布原因、风险说明、验证说明、回滚说明。
- preflight block 时不能创建或确认真实发布。
- warning 可继续，但必须可见。

### 4.4 发布单详情

发布单详情是核心页面。

信息区块：

- 基本信息。
- preflight 结果。
- 策略和确认状态。
- 发布记录摘要。
- 服务器日志入口。
- 事件流。
- 回滚和重新发布入口。

主要动作：

- 本人确认。
- 管理员确认。
- 驳回。
- queued 前取消。
- 创建回滚发布单。
- 重新发布。

状态提示：

- `pending_confirm`：显示需要谁确认。
- `queued`：显示排队中。
- `running`：显示执行中和刷新状态。
- `success`：显示成功。
- `failed`：显示失败原因和恢复入口。
- `partial`：必须显示部分成功数量，并按失败处理。

### 4.5 发布记录

目标：展示真实执行历史。

列表字段：

- 发布记录号。
- 服务。
- 环境。
- 版本。
- 状态。
- 成功/失败/跳过服务器数量。
- 开始时间。
- 结束时间。
- 耗时。
- 错误摘要。

详情：

- 执行快照。
- 服务器日志。
- 错误摘要。
- 关联发布单。

### 4.6 服务与版本

目标：维护发布对象。

页面内容：

- 项目列表。
- 服务列表。
- 服务详情。
- 版本列表。
- 版本注册。

要求：

- 服务详情展示可用环境、历史版本和部署目标状态。
- artifact URL 等敏感内容展示时脱敏。

### 4.7 环境与服务器

目标：维护运行目标。

页面内容：

- 环境列表，明确 `is_production`。
- 服务器列表。
- 服务器组。
- SSH 连接测试。

要求：

- 生产环境必须有清晰标识。
- 服务器连接配置完整性可见。
- 最近连接测试状态可见。

### 4.8 部署目标

目标：配置服务在环境中的执行方式。

字段：

- 服务。
- 环境。
- 执行器类型：mock / ssh。
- 运行目标：服务器 / 服务器组。
- 脚本路径。
- 工作目录。
- 环境变量。
- 超时。

### 4.9 发布策略

目标：配置门禁和冻结。

内容：

- 系统默认策略。
- 环境级策略。
- 服务级策略。
- 最终生效策略预览。
- 手动冻结开关。

要求：

- 生产管理员确认约束必须可见。
- 服务级策略不能让生产发布绕过管理员确认。
- 冻结打开后页面提示 queued 和 running 语义。

## 5. P1 页面

### 5.1 访问密钥管理

内容：

- 我的访问密钥。
- 管理员集成访问密钥。
- scope。
- 过期时间。
- 启用/禁用。
- 删除。

要求：

- 列表只显示 key 前缀。
- 创建后明文只显示一次。

### 5.2 通知配置

内容：

- 企业微信机器人 webhook 配置。
- 测试发送。
- 通知发送记录。

要求：

- webhook 不回显明文。
- 通知失败不影响主流程。

## 6. 状态视觉规则

| 状态 | 语义 | 展示 |
|------|------|------|
| `pending_confirm` | 待确认 | 黄色标签，显示确认人类型 |
| `queued` | 排队中 | 蓝色标签 |
| `running` | 执行中 | 蓝色动效或刷新提示 |
| `success` | 成功 | 绿色标签 |
| `failed` | 失败 | 红色标签，显示错误摘要 |
| `partial` | 部分成功 | 橙色标签，按失败处理 |
| `skipped` | 跳过 | 灰色标签 |

## 7. 日志和轮询

- 发布单详情在 queued/running 时可轮询。
- 轮询间隔应可控，避免过度请求。
- 用户应可手动刷新。
- 日志展示支持按服务器切换。
- 长日志第一版可分页或截断展示，并保留完整日志引用。

## 8. 空状态和错误状态

- 无发布单时给出创建入口。
- preflight block 必须展示阻断原因。
- warning 必须可见，但不阻止继续。
- SSH 连接失败展示错误分类和建议检查项。
- API 错误展示可读中文说明。

## 9. 验证要求

- 用户能通过 Web 创建发布单、preflight、确认、执行、查看日志。
- 管理员能配置环境、服务器、部署目标和策略。
- `partial` 和“混合版本”展示清晰。
- 页面不出现敏感凭据明文。
