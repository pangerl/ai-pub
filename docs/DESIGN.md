# 前端视觉规范

本文是 ai-pub Web 前端的视觉系统基线。它只描述当前产品界面需要长期遵守的颜色、排版、布局和组件状态，不记录阶段方案或营销页样式。

## 1. 视觉目标

- 工作台和发布中心应安静、清晰、便于扫描，优先服务重复操作。
- 管理配置页应强调对象关系和当前状态，不做装饰性大屏。
- 危险、阻断和待确认状态必须比普通信息更醒目。
- 第一版使用中文界面，技术标识、ID、scope、状态值保持原始英文。

## 2. 色彩

| 用途 | 颜色 |
| --- | --- |
| 主文字 / 主按钮 | `#171717` |
| 次级文字 | `#4d4d4d` |
| 弱提示 | `#888888` |
| 边框 | `#ebebeb` |
| 页面背景 | `#ffffff` |
| 弱背景 | `#fafafa` |
| 链接 / 成功强调 | `#0070f3` |
| 链接深色 | `#0761d1` |
| 错误 | `#c50000` |
| 错误背景 | `#f7d4d6` |
| 警告 | `#ab570a` |
| 警告背景 | `#ffefcf` |
| 成功浅背景 | `#aaffec` |

使用原则：

- 主按钮使用黑底白字。
- 状态标签使用浅色背景，不使用高饱和大面积色块。
- 渐变只用于登录页、引导卡片或少量强调区，不作为普通区块背景。

## 3. 排版

- 字体族：`Geist, Inter, ui-sans-serif, system-ui, -apple-system, BlinkMacSystemFont, "Segoe UI", sans-serif`。
- ID、版本、scope、事件类型使用等宽字体：`"Geist Mono", ui-monospace, SFMono-Regular, Menlo, monospace`。
- 页面标题使用 30-48px，紧凑行高，字重 600。
- 卡片和面板标题使用 15-26px，避免在工具界面里使用营销型大标题。
- 正文使用 14-16px，列表辅助文本使用 12px。

## 4. 布局

- 应用最大宽度为 `1440px`，主内容左右留白桌面端 `32px`、移动端 `16px`。
- 顶部导航固定在页面顶部，高度约 `64px`。
- 重复信息使用列表、表格化行或紧凑卡片；避免卡片套卡片。
- 发布详情、配置页和系统页可使用左侧分组导航 + 右侧工作区。
- 移动端在 `760px` 以下改为单列，横向导航允许滚动。

## 5. 组件

### 按钮

- 主要操作使用 Ant Design primary button，并统一覆盖为黑底。
- 次要操作使用默认按钮或文本按钮。
- 危险操作使用 danger 状态，动作文案必须明确。

### 面板

- 通用面板使用 `1px #ebebeb` 边框、`8px` 圆角、白底。
- 列表面板内部用分隔线，不额外给每行加重阴影。
- 引导面板可用黑底或极浅渐变，但必须服务任务入口。

### 状态

- `pending_confirm`：强调待确认人和下一步动作。
- `queued` / `running`：强调排队或执行中，提供刷新后的可见进度。
- `success`：使用蓝色或浅绿色状态，不制造强烈庆祝视觉。
- `failed` / `partial`：按失败处理，突出错误摘要、失败服务器和恢复入口。
- `skipped`：作为 fail-fast 后的未执行结果展示，不与成功混淆。

### 表单

- 表单默认纵向布局。
- 创建和编辑尽量使用 Drawer，不跳出当前工作区。
- 密钥、密码、webhook、私钥等敏感输入创建后不回显。

## 6. 页面规则

- 工作台只展示个人待办和最近需要处理的发布摘要，不承载全量查询。
- 发布中心承载完整发布单列表、筛选和创建入口。
- 发布详情是确认、驳回、取消、重试、回滚和查看事件的主操作区。
- 发布记录用于执行历史和服务器日志，不承担发布单审批动作。
- 配置页围绕“应用与版本、运行环境、部署连接”组织。
- 系统页承载用户、集成访问密钥、通知和凭据等管理员操作。

## 7. 可访问性与响应式

- 所有按钮和表单控件必须有可读文本或明确图标含义。
- 文本不得溢出按钮、标签、列表行和卡片；长 ID、URL、错误信息允许换行或省略。
- 列表在移动端隐藏次要列，但保留状态、对象名和下一步动作。
- 颜色不能作为唯一状态提示，必须配合文本。
